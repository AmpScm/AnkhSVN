Reimplement Ankh as a VS source control package
Rewrite the integration parts of AnkhSVN to use the new source control interfaces provided in VS 2005. This will give us far better integration than we have today, and most of the existing code can still be used, while some of the most buggy code we have today can be removed completely (solution explorer integration, for one). A question here would be whether to refactor the code so we can have two versions: one for downlevel versions such as 2002 and 2003 based on the addin APIs, and one for 2005 and upwards based on VSIP.
