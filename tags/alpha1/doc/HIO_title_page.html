Abstract
The development of the Subversion addin for Visual Studio .NET, named Ankh, was conducted as a final year project at HiO in cooperation with CollabNet.

Before Ankh was developed Subversion (SVN) had to be used from the command prompt. This is a very common approach for UNIX developers, but developers on Windows are more accustomed to graphical clients. The dominant developer tool on Windows is currently Microsoft Visual Studio.NET, and integration into this environment would make Subversion a viable alternative to a large number of Windows developers.

First step was to make a .NET wrapper for the Subversion's client API, and thereby use the wrapper to create a Visual Studio .NET addin for SVN. The system was coded in the C# and Managed C++ programming languages.

Improvements due to the development of Ankh:
· Most of the Windows users are not fond of using command prompts. By creating a GUI, SVN is made available for a huge amount of new Windows programmers. Today the audience of SVN are mainly Unix users. We hope to reach a new audience and convert the trend of UNIX user dominance.
· SVN is accessed directly from within VS.NET itself.
· Increased productivity since revision control is directly accessible in VS.NET.
· We can offer the Windows users a FREE open source revision control tool integrated in VS.NET.
