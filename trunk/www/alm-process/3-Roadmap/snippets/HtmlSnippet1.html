Discuss the roadmap in the AnkhSVN discussion forum .


We welcome anybody developing one of the below features and submitting a patch. Before you do, we'd appreciate it if you get in touch with us first to discuss the implementation. Send an email to dev@ankhsvn.open.collab.net .

AnkSVN Roadmap        
UI Cleanup Clean up the dialogs, get them more professional-looking and consistent in terms of look-and-feel. 2 30 Damien
Better Log support Ankh support for SVN log should be completely rewritten; move away from the HTML-based view to one based on a toolwindow with a listview. Support diffing between arbitrary revisions of a file. 2 40 ?
Orcas support Have the installer create the necessary registry keys for Orcas; test extensively whether Orcas breaks anything. 1 4 Arild/Sander
Log from repository explorer Log should be supported also in the Repository Explorer 2 8 ?
Reimplement Ankh as a VS source control package Rewrite the integration parts of AnkhSVN to use the new source control interfaces provided in VS 2005. This will give us far better integration than we have today, and most of the existing code can still be used, while some of the most buggy code we have today can be removed completely (solution explorer integration, for one). A question here would be whether to refactor the code so we can have two versions: one for downlevel versions such as 2002 and 2003 based on the addin APIs, and one for 2005 and upwards based on VSIP. 2 100 Arild/Sander
Support for editing svn properties Ankh currently has no way of editing SVN properties (svn:ignore et al) 2 10 ?
Merge support We should support svn merge, including the new merge tracking coming in svn 1.5 2 50 Arild
Sourceserver indexing Use Sourceserver (http://code.google.com/p/sourceserversharp/) to annotate our PDBs with information on how to retrieve the source on demand while debugging (this should probably be tied into the same process that provides dev libs (see below)). 2 10 Sander/Jeremy
bugtraq integration http://tortoisesvn.net/issuetracker_integration 1 40 ?
Better keyboard support Have Alt-Key combinations for all widgets in all dialogs. (Note, this is almost complete) 1 5 Arild
Localization support Allow localization of the UI. 3 50 ?
Better blame It should be possible to go to view log for revisions listed per line. 2 20 ?
Provide current developer libraries Building Ankh should be easier for developers to start hacking on Ankh. We have to provide pre-built zips of all of Ankh’s dependencies (the ones we do not keep in the repository). These should ideally be built regularly (and automatically) and uploaded (possibly alongside regular development builds of Ankh itself). 1 30 Jeremy
Distribute NSvn separately This will hopefully build more of an ecosystem around Ankh. 3 10 Jeremy
Migrating the repository Host the repository on OpenCollab 1   Arild/Guido
Migrating issue tracker Migrate existing issues to OpenCollab 1   Guido
Migrating download section Migrate downloads to OpenCollab 1   Guido
Allow icon sets Allow users to choose icon sets like TortoiseSVN 3 20 Damien
End-user documentation Supply end-user documentation 1   Arild/Sander/?
