**AnkhSVN source code**

Usage of the Ankh build environment requires at least:

  * Visual Studio 2008 (or 2005) Standard Edition or Higher

  * Visual Studio 2008 SDK (or Visual Studio 2005 SDK (4.0) for VS 2005)

  * A subversion client (e.g. AnkhSVN itself)

  * When using the VS 2005 SDK administrative permissions are required
  

And if you wish to build your own installers:

  * Votive (Including Wix 3.0) beta


Now you are ready to check out Ankh itself. Type the following:

    svn checkout http://ankhsvn.open.collab.net/svn/ankhsvn/trunk/ ankhsvn

Open and build AnkhSvn.2008.sln (or AnkhSvn.2005.sln if you are using Visual
Studio 2005).

Whey you build the AnkhSvn project inside the solution the AnkhSvn Package will be 
registered in the so called 'Experimental Hive' (a special test environment
of your Visual Studio installation).

You can start Visual Studio with the experimental hive if you select the
'Start Microsoft Visual Studio 2008 under Experimental hive' icon below your
VS SDK group in the start menu. But it is easier to copy these settings to the
debug command of the AnkhSvn project.

By default this will be something like:

VS2008: 
"%programfiles%\Microsoft Visual Studio 9.0\Common7\IDE\devenv.exe" /rootSuffix Exp /RANU


VS2005:
"%programfiles%\Microsoft Visual Studio 8\Common7\IDE\devenv.exe" /rootSuffix Exp


To register the package in your real Visual Studio you can use:

VS2008:
"%programfiles%\Microsoft Visual Studio 2008 SDK\VisualStudioIntegration\Tools\Bin\RegPkg.exe" src\AnkhSvn\bin\release\AnkhSvn.dll /codebase
"%programfiles%\Microsoft Visual Studio 9.0\Common7\IDE\devenv.com" /rt "Software\Microsoft\VisualStudio\9.0Exp" /RANU /setup

VS2005:
"%programfiles%\Microsoft Visual Studio 2005 SDK\2007.02\VisualStudioIntegration\Tools\Bin\RegPkg.exe" src\AnkhSvn\bin\release\AnkhSvn.dll /codebase
"%programfiles%\Microsoft Visual Studio 9.0\Common7\IDE\devenv.com" /rt "Software\Microsoft\VisualStudio\8.0Exp" /setup


(To remove the package from your Visual Studio you can replace /codebase with /unregister)
