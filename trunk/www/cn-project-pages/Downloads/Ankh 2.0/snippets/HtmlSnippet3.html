New features:

  • Built on Subversion 1.5.6.
  • Patch creation and applying (apply requires TortoiseSvn to be installed) (issue 497, issue 498).
  • Branching.
  • Standard support and detection for diff/merge tools:
    • Araxis Merge.
    • Beyond Compare.
    • Ellié Computing Merge.
    • KDiff3.
    • Perforce Visual Merge.
    • SourceGear DiffMerge.
    • TortoiseMerge.
    • WinMerge.
  • Relocate when switching detects it a relocate should happen (issue 544).
  • Integrated Annotate viewer.
  • Diffing files is a lot faster.
  • Overall performance improvements.
  • Track history when using "Save As".
  • Multiple working copy support. This means you can now use AnkhSVN to add solution without a common root to Subversion.
  • File -> Open -> Subversion Project can now open Projects (previously just Solutions).
  • Unified diff back.
  • Experimental support of Visual Studio 2010.
  • Subversion sub menu in the context menu on the editor.
  • Recent Changes tab, this is like the Synchronize View in Eclipse.
  • Submenus on projects and solutions that allow operations on the project/solution file. (The operations outside the submenu target the project/solution recursively).
  • Show/edit revision properties (issue 472).
  • Commit Dialog should remember keep log message history. (issue 542).
  • Resolve case conflict option.
  • Move to change list options & ignore-on-commit default change list.
  • Enabled showing property changes.
  • Partial bugtraq support; users can now enter bugid with commit.
  • Option to clear cached authentication data.

Most important bugfixes:

  • Unmodified check changed from green to blue, to help users with color blindness.
  • Reduced chance of conflicts when merging.
  • Resolved error when editing conflicts without a merge tool set.
  • "Revert changes" fixed (resulted in a NO-OP).
  • Fixed: ArgumentNullException when editting conflicts without a merge tool.
  • Fixed: problems where the focus is stolen (when creating new folders, or editing files).
  • Fixed: ArgumentNullException in Log Viewer (issue 456).
  • Fixed: NullReferenceException in Property Editor (issue 528).
  • Fixed: Directory rename in Website doesn't add/move files (issue 525).
  • Fixed: Project rename doesn't update solution file status (issue 531).
  • Fixed: Double click on Repository Explorer doesn't execute default action (issue 520).
  • Fixed: Merge Project Window Suddenly Close After Click Next (issue 537).
  • Fixed: Miscellaneous Files folder always shown (issue 545).
  • Fixed: Provide feedback when URI is inaccessible in ReposExplorer (issue 432).
  • Fixed: Visual studio crashes with ASP.NET MVC projects (issue 553).
  • Fixed: Problems with the F# project type.
